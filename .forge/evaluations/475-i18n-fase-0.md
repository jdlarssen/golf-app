## Risk notes for the owner (deploy-time)

- **rootParams is `experimental`.** `next.config.ts` enables
  `experimental.rootParams: true`. It builds green locally on Next 16.2.6 and is
  the documented pattern, but it is an experimental flag — watch the first Vercel
  prod build log to confirm the same 81 ◐ routes prerender there (the #538 lesson:
  PPR attribution can differ; use `--debug-prerender` if a route flips to ƒ).
- **users.locale cross-device staleness** is a documented, accepted limitation:
  a device with an existing NEXT_LOCALE cookie won't pick up a `users.locale`
  change made on another device until the cookie refreshes. The Phase 1 Profil
  toggle (updates both DB + cookie) keeps same-device behavior exact. No live
  e2e of the DB-override exists because no UI can set the column until Phase 1 —
  code-level evidence (Type A tests + proxy wiring) is sufficient for Phase 0.
- **Timezone pinned to `Europe/Oslo`** in `i18n/request.ts` to stop next-intl
  serializing the server's TZ (UTC on Vercel) into the client provider. Correct
  fix; worth a sanity glance at a date-heavy page in prod after deploy.
- **Migration 0093 already applied to prod** but the column is inert until this
  branch deploys (no code reads it pre-merge) — no ordering hazard.
- Benign build warning about multiple lockfiles (worktree + main repo) is a
  workspace-root inference notice, unrelated to this work.
